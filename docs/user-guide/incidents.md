# Incidents Guide

## Что Такое Инцидент

Инцидент — это подтверждённая техническая проблема или риск: сайт недоступен, SSL скоро истечёт, диск заканчивается, бэкап устарел, agents зависли.

## Severity

| Severity | Meaning |
| --- | --- |
| `info` | Информационное событие |
| `warning` | Риск требует внимания |
| `critical` | Проблема влияет на доступность или безопасность |

## Как Обрабатывать

1. Откройте карточку инцидента.
2. Проверьте evidence и affected check.
3. Нажмите acknowledge, если взяли в работу.
4. Исправьте проблему на сайте или инфраструктуре.
5. Дождитесь auto-resolve или закройте вручную с комментарием.

## Mute

Mute используется, если проблема известна и временно не требует уведомлений.

## Окно обслуживания

Окно обслуживания — режим «плановые работы»: обновление Bitrix, перенос, работы на сервере, когда кратковременные сбои ожидаемы.

Пока окно активно:

- новые инциденты по выбранным проверкам (или по всем) не создаются;
- повторные critical-напоминания в Telegram не отправляются;
- уже открытые инциденты остаются; при восстановлении закрываются как обычно;
- проверки продолжают выполняться, данные сохраняются.

Настраивается на карточке сайта в блоке «Окно обслуживания».

## Пороги проверок

На карточке сайта в блоке «Проверки» можно изменить пороги предупреждения и критичности (диск, бэкап, agents, SSL, домен, обновления модулей). Uptime и связь с модулем используют отдельные правила сервиса.
