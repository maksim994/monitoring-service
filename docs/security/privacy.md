# Privacy Rules

## Базовое Правило

Monitoring Service собирает только технические данные о состоянии сайта. Сбор персональных данных и бизнес-контента запрещён по умолчанию.

## Запрещённый Сбор

Не собирать и не передавать:

- персональные данные пользователей сайта;
- содержимое заказов;
- содержимое форм;
- дампы БД;
- файлы из `/upload/`;
- имена файлов, если они могут раскрыть данные клиента;
- cookie;
- session ids;
- Authorization headers;
- лицензионные ключи Bitrix;
- API secrets.

## Допустимые Данные

Допустимы:

- домен и URL сайта;
- версии PHP, Bitrix и модулей;
- технические метрики диска;
- возраст бэкапа;
- количество agents и lag;
- агрегированные статусы collectors;
- sanitized errors.

## UI Требование

В кабинете должна быть страница "Передаваемые данные" со списком payload-ов и объяснением, зачем каждый тип данных нужен.

## Logs

Логи backend и модуля не должны содержать:

- raw request body с потенциальными секретами;
- headers с secret/signature;
- персональные данные;
- webhook secrets;
- bot tokens.

## Retention

Retention зависит от тарифа. Пользователь должен видеть срок хранения истории в разделе тарифа.

## Self-hosted

Self-hosted режим не отменяет privacy rules. Он только меняет место хранения данных.
