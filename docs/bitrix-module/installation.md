# Bitrix Module Installation

## Назначение

Модуль `vendor.monitoring` устанавливается на сайт 1С-Битрикс или коробочный портал Битрикс24, собирает технические метрики и отправляет их в SaaS/self-hosted backend.

## Предварительные Требования

- Административный доступ к Bitrix.
- PHP с поддержкой `curl` или доступным `Bitrix\Main\Web\HttpClient`.
- Доступ сайта к API URL сервиса по HTTPS.
- Сгенерированные в кабинете `site_id` и `api_secret`.

## Ручная Установка Для MVP

1. Скопировать каталог модуля в:

```text
/local/modules/vendor.monitoring/
```

2. Открыть административный раздел Bitrix.
3. Перейти в Marketplace или список модулей.
4. Найти модуль Monitoring Service.
5. Нажать "Установить".
6. Перейти в настройки модуля.

## Настройка Подключения

Заполнить поля:

| Поле | Значение |
| --- | --- |
| Mode | `saas` или `self_hosted` |
| API URL | Базовый URL сервиса, **без** `/api` (модуль сам добавляет `/api/v1/...`) |
| Site ID | Значение из кабинета |
| API Secret | Секрет, показанный при создании сайта |

После сохранения:

1. Нажать "Проверить подключение".
2. Нажать "Отправить тестовый heartbeat".
3. Убедиться, что в кабинете сайт перешёл в статус connected/ok.

## Агенты И Cron

Основной агент:

```php
\Vendor\Monitoring\Cli\Agent\CollectorAgent::run();
```

Интервал MVP: 300 секунд.

Production-рекомендация: перевести Bitrix agents на cron, чтобы сбор метрик не выполнялся на пользовательских хитах.

## Удаление

При удалении модуль должен:

- удалить зарегистрированные agents;
- удалить обработчики событий;
- предложить оставить или удалить локальные таблицы;
- удалить options при полном удалении;
- вызвать `ModuleManager::unRegisterModule`.

## Проверка Успешной Установки

Установка считается успешной, если:

- модуль отображается как установленный;
- options page открывается без ошибок;
- test connection возвращает `ok`;
- heartbeat появляется в кабинете в течение 2 минут;
- локальная очередь не содержит постоянных failed-записей.

## Типовые Ошибки

| Ошибка | Возможная причина | Действие |
| --- | --- | --- |
| Invalid signature | Неверный secret или рассинхрон времени | Пересоздать ключ, проверить время сервера |
| API unavailable | Сеть, firewall, DNS | Проверить доступность API URL с сервера |
| Payload too large | Collector собрал слишком много данных | Уменьшить лимиты remote config |
| Agent not running | Agents на хитах не выполняются | Настроить cron |
