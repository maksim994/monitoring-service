# Transmitted Data

## Принцип

По умолчанию Bitrix-модуль передаёт только технические данные, необходимые для мониторинга работоспособности сайта. Модуль не должен собирать персональные данные пользователей сайта, содержимое заказов, форм, файлов, cookie или сессий.

## Передаётся В SaaS/Self-hosted

### Идентификация Сайта

- `site_id`;
- домен;
- canonical site URL;
- версия модуля;
- режим работы: `saas`, `self_hosted`, `module_only`.

### Environment

- версия PHP;
- версия Bitrix main;
- кодировка;
- timezone;
- server software;
- значения PHP limits;
- наличие ключевых PHP extensions.

### Disk

- свободное место в байтах;
- процент свободного места;
- агрегированный размер `/upload/`, если доступно безопасно;
- агрегированный размер cache-директорий, если доступно безопасно.

### Backup

- дата последнего успешного бэкапа;
- возраст бэкапа;
- статус: ok, stale, missing, unknown.

### Modules

- список установленных модулей;
- версии модулей;
- количество доступных обновлений, если доступно безопасно;
- дата последней проверки обновлений.

### Agents

- количество активных agents;
- количество просроченных agents;
- максимальный lag;
- технический идентификатор зависшего agent без чувствительных аргументов.

### Diagnostics

- ошибки collectors в sanitized виде;
- состояние локальной очереди;
- дата последней успешной отправки;
- config version.

## Не Передаётся

- ФИО, телефоны, email пользователей сайта.
- Содержимое заказов.
- Содержимое web forms.
- Дампы БД.
- Файлы и имена файлов из `/upload/`, если они могут раскрывать данные клиента.
- Cookie.
- PHP session ids.
- Authorization headers.
- Лицензионные ключи Bitrix.
- API secret модуля.
- Полные raw payload logs по умолчанию.

## Хранение Секретов

- `api_secret` хранится в настройках модуля только в зашифрованном виде.
- В интерфейсе secret отображается только маской.
- В логах secret не отображается никогда.

## Объяснение Для Клиента

Monitoring Service собирает технические признаки состояния сайта: доступность, версии, свободное место, свежесть бэкапов, состояние agents и обновлений. Эти данные нужны, чтобы заранее предупредить о сбое или риске. Сервис не анализирует бизнес-данные сайта и не забирает пользовательский контент.
